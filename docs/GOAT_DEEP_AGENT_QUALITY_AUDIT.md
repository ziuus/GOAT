# GOAT Deep Agent Quality Audit (Phase 7.0)

## Overview
Phase 7.0 focuses on transitioning the Prime Agents (Cofounder, Learner, Researcher, and Builder) from basic mock/stub-driven workflows into deeply integrated, fully autonomous entities. They will utilize Brain context packs, provider routing, and actual LLM generation via `PromptForge` and `AgentRuntime`.

## Current State Analysis

### 1. Cofounder Agent (`src/agents/cofounder.rs`)
- **Status:** Uses mocked structures and hardcoded responses (e.g., `generate_scorecard` returns a static 33 total score, `generate_validation_plan` returns fixed steps).
- **Missing Integration:** Does not query the Brain for past ideas, does not use the LLM to actually score or validate, does not use `ProviderRouting` to select models.
- **Action Plan:** Connect `generate_*` methods to the `AgentRuntime` and `PromptForge`. Fetch competitor analysis from the Brain. Route strategic tasks to high-tier reasoning models.

### 2. Learner Agent (`src/agents/learner.rs`)
- **Status:** Similar to Cofounder, returns hardcoded JSON mock data for goals, roadmaps, and progress reports.
- **Missing Integration:** Does not assess complexity or adapt to the user's actual knowledge level using the Brain context.
- **Action Plan:** Implement dynamic roadmap generation via the LLM. Read user's existing knowledge from `brain_context.rs` to inform curriculum depth. Route learning tasks to fast models, except for complex topic breakdown.

### 3. Researcher Agent (`src/agents/researcher.rs`)
- **Status:** Returns empty mock structs (`ResearchBrief`, `ResearchPlan`).
- **Missing Integration:** No real search integration or evidence synthesis.
- **Action Plan:** Equip Researcher with `ProviderRouting` to handle complex synthesis. Link to Brain context packs to avoid re-researching known topics. Create real LLM prompts for generating Briefs and Reports.

### 4. Builder Agent (`src/agents/builder.rs` - Currently Missing)
- **Status:** Registered in `registry.rs` but lacks a dedicated implementation file in `src/agents/`.
- **Missing Integration:** Not implemented. 
- **Action Plan:** Create `src/agents/builder.rs`. Implement a state machine for feature implementation, code review requests, and testing. Route code generation to coding-specific models.

## Architecture Upgrades

### `src/agent_quality.rs`
We will introduce a shared helper module to standardize how Prime Agents interact with the rest of the system:
- **`QualityGate`:** Evaluate generated content before saving it (e.g., verifying a scorecard has valid values).
- **`ContextPacker`:** Automatically assemble relevant Brain memories and Timeline events for a given Prime Agent.

### Provider Routing Integration
Agent workflows will specify `TaskKind` (e.g., `Strategic`, `Coding`, `Synthesis`) so the runtime uses the optimal model.

### Runtime Job Templates
Job templates in `PromptForge` or `AgentRuntime` will be expanded to support multi-step deep agent workflows (e.g., Researcher generating a plan -> sourcing -> drafting).

### Dashboard Integration
Ensure that reports generated by these upgraded agents use `ReportManager` properly, producing markdown that is displayed beautifully in the TUI and Web Dashboard.
