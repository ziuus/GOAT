# GOAT Browser UI QA (Phase 6.9)

## 1. Overview
The UI QA workflow is designed to execute standard interface checks against targets like local projects or staging websites.

## 2. Steps Performed
1. **Open URL**: Load target endpoint.
2. **Capture Screenshot**: Take a visual capture.
3. **Inspect DOM**: Extract heading counts, CTAs, forms, buttons.
4. **Check Links**: Audit broken URLs or accessibility details.
5. **Check Accessibility Risks**: Flag forms without labels or images without alt text.
6. **Generate QA Report**: Store results as an artifact.
