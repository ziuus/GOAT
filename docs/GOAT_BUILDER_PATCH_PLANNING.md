# GOAT Builder Patch Planning

Patch planning models the specific code modifications intended to resolve a user request.

## Safety & Planning Constraints
- **Affected Files:** A strict inventory of files targeted for change.
- **Rollback Plan:** Step-by-step description of how to restore files or rollback git state.
- **Risk Classification:** Classify changes (Low, Medium, High) to drive user approval.
- **Acceptance Criteria:** Specific compile/test benchmarks to verify change completion.
