# GOAT v0.1.0-alpha.1 — First Alpha Release

Welcome to the very first alpha release of GOAT (General Objective Agentic Task-engine)! This milestone transitions GOAT from a concept to a functional, locally-hosted, migration-grade AI agent foundation.

## What Works (Core Features Included)
* **Local-First Brain:** SQLite-backed memory preserving missions, workflows, and checkpoints between sessions.
* **Project Intelligence:** `goat learn` parses AST structures to build deep, intelligent repository maps (`repo-map`).
* **Safe Patch Flow:** Code changes are proposed as diffs, analyzed by the Context-Aware Diff Analyzer, and explicitly approved before altering the workspace.
* **ApprovalGate:** Every execution—from terminal commands to filesystem writes—is strictly gated by default to ensure safety.
* **Validation Runner:** A built-in validation engine (`goat validate`) that verifies changes natively via `cargo test`, `npm test`, etc.
* **Capability & Skill Registry:** An extensible architecture supporting repeatable AI workflows without running blind shell scripts. Includes MCP (Model Context Protocol) auto-start prevention.
* **Interactive TUI:** A full-featured terminal interface with `/tools`, `/skills`, and `/mission` dashboards.

## The Safety Model (Why GOAT is Different)
GOAT is fundamentally **fail-closed**.
Tools like Aider or Claude Code often optimize for zero-click autonomy, running raw shell commands directly. GOAT assumes that LLMs can make mistakes. All capabilities, skills, and terminal commands must be prepared, listed in the Capability Registry, and explicitly pass the `ApprovalGate` before running. 

If it's not explicitly approved, it doesn't execute.

## Migration Notes (For Users Coming from Other Tools)
* **From Claude Code / Codex CLI:** You'll notice GOAT interrupts you more often for approval. This is by design. Read our [Alpha Quickstart](../GOAT_ALPHA_QUICKSTART.md).
* **From Aider:** GOAT doesn't auto-commit. Changes are staged as patches (`goat patch propose`) and must be applied via `goat patch apply`.
* **From Cline/Cursor:** GOAT is CLI-first right now. It does not actively sync with your unsaved IDE tabs.

## Known Limitations
* **IDE Context:** Lacking native VS Code/IntelliJ extensions to read real-time editor state.
* **Approval Fatigue:** Running complex builds can trigger multiple approval prompts. Future versions will introduce tiered Approval Profiles.
* **Advanced Capability Scheduling:** Cron-like autonomous agent execution is delayed until post-alpha.

## Installation & Testing
Run the following from the source directory to build the release binary:
```bash
bash scripts/build-release.sh
```
Or directly via cargo:
```bash
cargo install --path .
```
Verify your installation:
```bash
bash scripts/smoke-test-alpha.sh
# or manually:
goat doctor alpha
```
