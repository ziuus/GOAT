# Release v0.14.0-alpha.1

**GOAT is officially entering Public Alpha.** This release solidifies the local-first multi-agent foundations, emphasizing workflow safety, inspection, and explicit user control.

## 🌟 Major Features
* **ApprovalGate & Safety-First Workflow:** Every destructive file system change or shell command requires user approval. Agents plan safely; humans authorize.
* **Six Deep Prime Agents:**
  * **Cofounder:** MVP scoring and validation.
  * **Researcher:** Evidence-grounded reports and competitor analysis.
  * **Socializer:** Safe distribution planning (drafts only, no auto-post).
  * **Designer:** UI/UX reviews and aesthetic auditing.
  * **Operator:** Incident logging and rollback planning (runbooks only, no auto-deploy).
  * **Builder:** Code planning with explicit fallback constraints.
* **Local-first Memory (Brain):** SQLite-backed vector search and Semantic indexing.
* **Interfaces:** A fully-featured Terminal UI (TUI) and Next.js Dashboard.

## 🛡️ Safety Guarantees
* GOAT cannot push to git, deploy, or rollback without ApprovalGate.
* External API integrations are completely opt-in.
* Secrets and environments are fully isolated per-project.

## 🚀 Quick Install
```bash
git clone https://github.com/ziuus/GOAT.git
cd GOAT
cargo build --release
cargo run --release -- setup
```

## 🔧 Verifying the Installation (Smoke Test)
Run the built-in smoke test script to verify environment constraints:
```bash
chmod +x scripts/smoke-alpha.sh
./scripts/smoke-alpha.sh
```

## ⚠️ Known Limitations
Please review [GOAT_KNOWN_LIMITATIONS.md](../GOAT_KNOWN_LIMITATIONS.md) before diving deep. Notably, provider integrations must be added manually in `~/.config/goat/goat.toml`, and browser automations require external dependencies.
