# GOAT GitHub Workflow

This document outlines the GitHub-native workflow integration built into GOAT (Phase 5.11).

## Architecture overview
GOAT integrates with GitHub using the `gh` CLI. It implements a local-first, approval-gated workflow designed for agentic interaction.
Tokens are never stored or logged by GOAT; instead, GOAT piggybacks onto the `gh auth status` authentication of the local user.

The `GitHubWorkflowManager` manages a state machine that tracks progress through the lifecycle of a typical open-source or enterprise pull request:
1. `Unlinked`
2. `IssueLinked`
3. `BranchCreated`
4. `DraftPrCreated`
5. `PrSubmitted`
6. `Completed`

## CLI Commands
The `/github` or `\gh` prefix is used for all GitHub workflow commands.

* `/github status` - Check the current workflow state.
* `/github doctor` - Check `gh` installation, remote status, and authentication.
* `/github issue link <id>` - Link the current session to a specific GitHub issue.
* `/github issue unlink` - Unlink the current session from an issue.
* `/github branch plan` - Generate a descriptive branch name based on the linked issue.
* `/github branch create` - Create the branch and check it out locally.
* `/github pr draft` - Generate a draft pull request title and body.
* `/github pr create` - Request to push the branch and open the PR (Requires ApprovalGate).

## Dashboard Integration
The new `/github` route in the dashboard provides a visual interface for managing the workflow:
* View and link context (Issues).
* Plan and generate branches.
* Preview PR Drafts (generated by aggregating timeline/session data).
* Security checking UI ensuring that code push operations are explicitly approved.

## Security
* **ApprovalGate**: Creating PRs and pushing branches require explicit user approval via `ApprovalGate`. 
* **Redaction**: All CLI outputs from `gh` are scrubbed before being logged or displayed to prevent token or PII leakage.
* **No Auto-Closing**: GOAT does not automatically close or resolve issues. It relies on GitHub's native `Fixes #...` PR syntax.
