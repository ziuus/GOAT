# GOAT Researcher Quality Audit

## Current State (Post Phase 7.4)

### What Researcher Can Do Now
The Researcher agent can create basic topics and briefs. It has rudimentary structures for `ResearchTopic`, `ResearchSource`, `ResearchEvidenceNote`, `ResearchCompetitorScan`, `ResearchTechnologyComparison`, and `ResearchBrief` inside `src/agents/researcher.rs`. 
It persists these to `~/.local/share/goat/agents/prime/researcher/` using JSONL files (`topics.jsonl`, `sources.jsonl`, etc.).

### What is Shallow/Simple
- **Source tracking:** It only saves URLs and text summaries. There is no rigorous metadata extraction, no hash-based deduplication, and no detailed provenance tracking.
- **Citations:** It has `key_claims` and `limitations` as strings, but no rigorous `ResearchCitation` model with grading or linking back to specific sources.
- **Evidence Grading:** There is a simple string `"low", "medium", "high"` for confidence in `ResearchEvidenceNote`, but no `EvidenceGrader` or `EvidenceGradeResult` capturing completeness, independent sources, or conflicts.
- **Browser Artifacts:** There is no integration for ingesting screenshots or DOM summaries generated by the browser tools.
- **Competitor Intelligence:** There's a basic `ResearchCompetitorScan`, but it lacks deeper matrix comparison integration or a structured report format tied to verifiable claims.
- **Brain Integration:** Documents are not indexed properly into the Brain.

### Phase 7.5 Scope
- **Source Management:** A new `src/research_sources.rs` module formalizing source taxonomy, trust levels, and provenance.
- **Citations:** A rigorous `ResearchCitation` model ensuring claims tie directly to evidence, eliminating fake citations.
- **Evidence Grading:** Structured `EvidenceGradeResult` taking source type, trust level, and recency into account.
- **Browser Artifact Integration:** Support for mapping browser-captured evidence into source material without aggressive web scraping.
- **Brain & API:** Full exposure of these rigorous models to the API and Dashboard.

### What Remains Partial
- Real-time deep web crawling is intentionally disabled to avoid aggressive scraping.
- Full ML-based cross-referencing of conflicting evidence is simulated using LLMs, but strict logical verification requires user review.
