# GOAT Report System

The Report System provides a structured output layer for agents, allowing them to present findings, plans, and analysis in a readable, persistent format rather than scrolling endlessly in the terminal.

## Core Concepts

1. **Report Manager:** The central registry handling generation, storage, and retrieval of reports.
2. **Report Template:** A defined structure indicating what sections a report must contain.
3. **Report Kind:** The category of the report (e.g., `founder_weekly_report`, `security_review_report`).
4. **Report Output:** The final Markdown content generated by the agent.

## Storage
Reports are generated as Markdown files and stored locally:
`~/.local/share/goat/reports/`

Each report includes metadata (in frontmatter or via a database link) tracking:
- Generating Agent ID
- Timestamp
- Source References (links to files, URLs, or timeline events)

## Common Report Kinds

- `founder_weekly_report`: Synthesizes progress, roadblocks, and next week's focus (generated by Cofounder).
- `validation_report`: Results of market research and competitor analysis.
- `social_content_plan`: A scheduled list of draft posts (generated by Socializer).
- `design_review_report`: UX/UI feedback against best practices.
- `research_brief`: A condensed summary of a technical or market topic.
- `build_status_report`: CI/CD health, recent commits, test coverage.

## Dashboard Integration
Reports are viewable in the React Dashboard under the `/reports` route (or the Reports tab inside `/agents`). 
Features include filtering by agent, viewing source references, and copying Markdown to the clipboard.

## Security & Privacy
Reports are stored entirely locally. They do not contain raw secrets; agents are instructed to redact API keys or passwords before writing report content.
