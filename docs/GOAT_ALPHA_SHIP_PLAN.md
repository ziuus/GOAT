# GOAT Alpha Ship Plan

## Priority List

### 1. Must work for alpha:
These features represent the core "Agent OS" promise and must be polished, error-free, and honest about their capabilities.
* **Setup / Doctor**: Clear CLI commands to verify environment.
* **Dashboard Home**: Working quick links, accurate system health, clear "Start Here" flow.
* **Agents Overview**: Command center view showing available and planned agents.
* **Learner**: Ability to view a learning roadmap (even if local-only demo).
* **Cofounder**: Ability to draft a startup idea and view the scorecard UI.
* **PromptForge**: UI for refining prompts.
* **Reports**: View generated reports.
* **Timeline**: View chronological project events.
* **README Quick Start**: Simple, working install instructions.

### 2. Nice to show (Experimental/Mocked):
These features exist in the UI to demonstrate the vision but lack deep backend wiring. They should be clearly labeled as "Experimental".
* **Socializer**: Ethical distribution UI shell.
* **Designer**: UI/UX review shell.
* **Researcher**: Brief generation shell.
* **Operator**: DevOps planning shell.
* **Brain Search / Memory Galaxy**: Vector search UI preview.

### 3. Experimental (Hidden or CLI-only):
These features are too raw for the primary Alpha dashboard flow and should not be prominently featured until stabilized.
* **Voice**: TTS/STT endpoints.
* **Transports**: Discord/Telegram bots.
* **Browser Automation**: Playwright hooks.
* **Deep Collaboration**: Multi-agent concurrent execution (accessible via CLI `/collab` but keep dashboard focus tight).
* **External Integrations**: Third-party API triggers.
